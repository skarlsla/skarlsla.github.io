Cursor AI는 강력한 AI 코딩 도우미로, 블로그 포스트 작성을 훨씬 효율적으로 만들어줍니다. 이번 포스트는 Cursor AI와의 대화를 통해 작성되었으며, 다음과 같은 과정으로 진행되었습니다:

Cursor AI 작성 과정

  1. 초기 요청: "JWT Authentication in Golang using Gin Web Framework" 포스트 작성 요청
  2. AI 분석:
    • 기존 블로그 구조 파악
    • 필요한 디렉토리 구조 확인
    • 포스트 템플릿 분석
  3. 포스트 생성:
    • 적절한 디렉토리 구조 생성 (2024/03/24/jwt-auth-gin/)
    • 메타데이터 자동 설정
    • HTML 구조에 맞는 콘텐츠 작성
  4. Git 작업:
    • 변경사항 확인 (git status)
    • 새 파일 추가 (git add)
    • 커밋 및 푸시 진행

  • 메타데이터 자동 생성
  • 마크다운과 HTML의 적절한 변환
  • 코드 블록의 자동 구문 강조
  • Git 명령어 실행 가이드

블로그 포스트 작성 과정

1. 디렉토리 구조 파악

GitHub Pages 블로그는 특정한 디렉토리 구조를 따릅니다. 새 포스트를 작성하기 전에 먼저 기존 블로그의 구조를 파악해야 합니다.

  • 연도별 디렉토리 구조 (예: 2024/03/24/)
  • 각 포스트는 자신만의 디렉토리를 가짐
  • 포스트 내용은 index.html 파일에 작성

2. 새 포스트 디렉토리 생성

3. 포스트 메타데이터 설정

HTML 파일의 head 섹션에 필요한 메타데이터를 설정합니다:

  • 제목 (title)
  • 설명 (description)
  • 키워드 (keywords)
  • 작성일 (datePublished)
  • 카테고리 (category)
  • 태그 (tags)

4. 포스트 내용 작성

포스트의 내용은 HTML 구조를 따르되, 마크다운 스타일의 가독성 좋은 형식으로 작성합니다:

  • 제목은 h1~h6 태그 사용
  • 코드 블록은 highlight 클래스를 가진 figure 태그 사용
  • 문단은 p 태그 사용
  • 목록은 ul/ol과 li 태그 사용

6. 배포 확인

GitHub Pages는 자동으로 변경사항을 감지하고 사이트를 다시 빌드합니다. 보통 몇 분 내에 새로운 포스트가 웹사이트에 반영됩니다.

    • 메인 애플리케이션

  • 보안 고려사항
  • 결론

주의사항

  1. 포스트 URL이 겹치지 않도록 고유한 디렉토리 이름 사용
  2. 모든 메타데이터 필드를 올바르게 설정
  3. 이미지를 사용할 경우 적절한 경로 설정
  4. 코드 블록은 적절한 언어 하이라이팅 적용
  5. 마크다운 문법과 HTML 구조의 적절한 조화

결론

GitHub Pages 블로그에 새로운 포스트를 작성하는 것은 HTML 구조를 이해하고 일관된 형식을 유지하는 것이 중요합니다. 위의 과정을 따르면 깔끔하고 체계적인 기술 블로그 포스트를 작성할 수 있습니다.
